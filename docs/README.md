# Metamsg

#### Q&A

+ 如何在一条流上保证消息请求和结果匹配？
    
    1. 请求-响应配对：当从客户端发送到服务器时，每个 gRPC 请求消息都标有一个唯一标识符，称为“调用 ID”或“序列号”。
       服务器按顺序处理请求，并使用此调用 ID 来跟踪哪个请求对应哪个响应。
    
    2. 流复用：gRPC 使用 HTTP/2 作为其传输协议，允许在单个连接中并发多个流。
       每个 gRPC 请求-响应对都与一个单独的流关联。
       这种多路复用允许在单个连接内交错多个请求和响应。
